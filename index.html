The three layers

Sources go in, knowledge comes out, and a set of rules keeps it consistent. That separation is what lets the wiki stay trustworthy as it grows.

raw/

Sources

Immutable source of truth — articles, papers, transcripts, data. The agent reads these and never edits them.

wiki/

Knowledge

LLM-generated, interlinked markdown across four areas. The agent owns this layer entirely: pages, cross-links, consistency.

CLAUDE.md

The rules

How the wiki is structured and maintained. Read every session, co-evolved as your conventions improve.

Quick start

Copy the folder, point your agent at it, and start feeding it sources.

  1. Copy this vaultGrab the whole folder, rename it, and optionally git init so everyone forks their own.
  2. Open it in Claude CodeThe agent reads CLAUDE.md at the start of every session — that's the contract.
  3. Drop a source into raw/An article, a paper, a transcript, notes, an image.
  4. Ask it to ingestIt summarizes the source, rates its credibility, and files cited, confidence-tagged claims into wiki/.
  5. Ask questionsGood answers get filed back as new pages — your explorations compound instead of evaporating.

What's new in this refresh

Sharper research methods so a whole cohort produces knowledge the same, trustworthy way. Full detail in METHODOLOGY.md.

A named research loop

Frame → Discover → Triangulate → Synthesize → File back → Review. Everyone researches identically.

Source credibility ratings

high / medium / low via a CRAAP checklist, with conflict-of-interest disclosed up front.

Confidence tags on claims

[C:high] / [C:med] / [C:low] — solid facts and hypotheses stay visibly distinct.

Triangulation rule

Load-bearing claims need ≥2 independent sources before a synthesis leans on them.

A live question backlog

questions.md keeps the vault driving forward between sessions instead of stalling.

Freshness discipline

review-by dates plus a lint pass so time-sensitive claims don't quietly rot.
