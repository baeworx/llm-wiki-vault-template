BAEWORXS Vault — LLM Wiki Cohort Guide

~5 min read · updated 2026-07-13 · version 2.0 · license MIT · built by Bae
A wiki your AI agent maintains for you — reading sources, extracting knowledge, and keeping an interlinked, trustworthy set of notes that compounds over time instead of being re-derived on every question. This guide is the operating reference for the crew.

⌁ Key Takeaways

  • Three layers: immutable raw/ sources → LLM-owned wiki/ knowledge → the CLAUDE.md ruleset.
  • Every claim is cited and confidence-tagged; load-bearing claims need ≥2 independent sources.
  • One repeatable research loop: Frame → Discover → Triangulate → Synthesize → File back → Review.
  • Copy the folder, point your agent at it, feed it sources. Knowledge compounds instead of evaporating.

01 // OVERVIEW

What this is

An LLM-maintained knowledge base. You curate sources and ask questions; your agent reads the sources, extracts knowledge, and maintains an interlinked wiki of markdown files. Sources go in, cited knowledge comes out, and a small ruleset keeps it consistent as it grows.

02 // ARCHITECTURE

The three layers

The separation is what lets the wiki stay trustworthy as it grows.

LayerPathRole
Sourcesraw/Immutable source of truth — articles, papers, transcripts, data. The agent reads these and never edits them.
Knowledgewiki/LLM-generated, interlinked markdown across four areas (research · personal · book · general). The agent owns this layer entirely.
RulesCLAUDE.mdHow the wiki is structured and maintained. Read every session, co-evolved as conventions improve.

03 // BOOT SEQUENCE

Quick start

  1. Copy this vault — grab the whole folder, rename it, optionally git init so everyone forks their own.
  2. Open it in Claude Code — the agent reads CLAUDE.md at the start of every session.
  3. Drop a source into raw/ — an article, a paper, a transcript, notes, an image.
  4. Ask it to ingest — it rates the source's credibility and files cited, confidence-tagged claims into wiki/.
  5. Ask questions — good answers get filed back as new pages, so explorations compound.

04 // METHOD

The research method

A claim in this wiki should be more trustworthy than the same claim pulled fresh from a search engine — because it's been evaluated, cited, cross-checked, and dated.

The loop

baeworxs@vault: ~/llm-wiki $ research --loop
FRAME  ›  DISCOVER  ›  TRIANGULATE  ›  SYNTHESIZE  ›  FILE BACK  ›  REVIEW
  │          │             │              │              │            │
  frame Q    prefer        corroborate    weigh          cite +       set review-by,
  in         primary       across ≥2      the            cross-link,  lint for
  questions  sources       independent    evidence       update index staleness

Source credibility — CRAAP

LensAskRed flag
CurrencyHow old? Fast-moving topic?Undated; stale on a moving target
RelevanceDoes it address the question?Tangential, cherry-picked
AuthorityWho made it? Standing?Anonymous, no track record
AccuracyEvidenced? Reproducible?No sources, "trust me"
PurposeWhy does it exist? Who profits?Selling something, affiliate agenda

Confidence tags

TagMeaning
[C:high]Corroborated across independent sources, or direct primary evidence.
[C:med]A single credible source; plausible but not cross-checked.
[C:low]A single weak/biased source, or an inference. Treat as a hypothesis.

Full detail in METHODOLOGY.md.
